Brief projet — Site vitrine : Café Pavillon (Café parisien, chic & branché)
Objectif
Créer un site vitrine one-page (avec sections ancrées) responsive et accessible qui reflète l’univers élégant et chaleureux d’un café parisien haut de gamme. Le site sert à : présenter le lieu, montrer la carte, inspirer (photos), faciliter la réservation et la privatisation, et améliorer la découverte locale (SEO local).
Direction artistique (instructions précises)
Ton & personnalité : élégant, chaleureux, discret, contemporain. Pas de clinquant. Atmosphère « salon parisien moderne » : lumière chaude, matériaux naturels (bois, cuir, céramique), touches terracotta.
Palette (valeurs hex exactes) :
Charcoal (texte principal) : #1F2937
Warm Sand (accent / boutons) : #C49B6E
Soft Cream (background) : #F6F3EE
Terracotta (call to action accent) : #A83E2E
White carte / surfaces : #FFFFFF
Typographie :
Titres : Playfair Display (weights 400,700) — serif, letter-spacing normal.
Texte / UI : Inter (300,400,600,700) — sans serif, variable for weight.
Fallbacks : serif / system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial.
Iconographie : lucide / feather style, stroke 1.5–2px, round caps.
Photographie : images lumineuses, palette chaude, profondeur de champ peu profonde (bokeh léger), gros plans sur cafés/pâtisseries, plans d’ambiance montrant clientèle discrète. Prioriser photos propre marque (studio/photographe) ; sinon Unsplash pour prototype.
Spacing & grid : large aérations (64px sur desktop entre sections), container centré max-width 1200px, card corners 12–16px, shadows légers box-shadow: 0 6px 20px rgba(31,41,55,0.08).
Animation : micro-animations subtiles — fade/slide-in pour hero, hover scale 1.03 sur images, button press animation (scale 0.98) ; durée 160–260ms, ease-out.

Header (sticky)
Logo (text+icon) à gauche, nav ancrée à droite (À propos, Menu, Galerie, Privatisations, Réserver, Contact).
CTA bouton « Réserver » visible en desktop et dans un petit bouton flottant sur mobile.
Hero
Grand visuel (image large, aspect 16:9 desktop), H1, sous-titre, 2 CTA (Voir le menu / Réserver), coordonnées rapides (adresse, téléphone, horaires).
Texte hero exact (voir section « Copy »).
À propos / Vision
Texte court, 2 bullets (grains sélectionnés, pâtisseries maison), photo d’accompagnement.
Carte / Menu — coups de coeur
Carte condensée, catégories, prix. Bouton « Voir le menu complet (PDF) ».
Galerie
Masonry ou grid 3 colonnes desktop, lazy loading, lightbox pour visualisation.

Footer
Mentions légales, horaires, réseaux, newsletter opt-in (double opt-in).

Comportement / Responsive / Accessibility
Breakpoints : mobile-first — sm: 640px, md: 768px, lg: 1024px, xl: 1280px.
Mobile :
Header compact avec logo + hamburger.
CTA « Réserver » accessible flottant (bottom right) sur mobile.
Hero image height réduite, texte centré.
Accessibilité :
Contraste minimal WCAG AA. Vérifier contraste texte/background (colors fournis sont conformes pour texte principal sur background clair ; vérifier boutons et éléments sur images).
Tous les éléments interactifs doivent avoir :focus visible, aria-label si nécessaire, semantic HTML.
Forms : labels liés for/id, aria-invalid pour erreurs, error messages annoncés.
Images : alt explicite. Gallery images alt="Photo — [description courte]".
Nav skip link to main content.
Keyboard accessible lightbox.
Use lang="fr" on html.
Internationalisation : site principal en français. Préparer architecture pour ajout d’une version anglaise (i18n keys).
Performance & assets
Images :
Fournir images optimisées : WebP + fallback JPEG. Serve responsive sizes (srcset, sizes) ; desktop up to 1600px, mobile 480px.
LQIP / blurred placeholder technique pour images hero/gallery (SVG or base64 tiny image).
Lazy-load non-critical images (loading="lazy").
Fonts : self-host recommended (woff2) to avoid CLS. Preload critical fonts; use font-display: swap.
Bundle : keep initial JS under 150–200KB gzipped if possible. Hydration only where nécessaire. Use code splitting for gallery/lightbox.
CDN : hébergement des assets statiques (images) via CDN (Cloudflare/Akamai/Netlify).
Performance budget : Lighthouse scores → aim >=90 perf, >=90 accessibility.

Intégrations techniques ( recommandations concrètes )
Réservation : idéalement intégrer Resy / OpenTable / Tock via widget. Si pas disponible -> form backend : POST /api/reservations -> envoi email + sauvegarde DB (example payload JSON plus bas).
Formulaire contact / devis : POST /api/contact -> envoi mail + sauvegarde.
Map : Leaflet + OpenStreetMap (pas de clé) ou Google Maps (clé → handle billing/GMaps API).
CMS (optionnel) : Sanity / Contentful / Strapi pour menu & images (prévoir modèles : MenuCategory, MenuItem, GalleryImage, Event).
Analytics / tracking : Google Analytics 4 + consent banner (CMP). Respect RGPD (consent before analytics). Event tracking : CTA clicks, reservation submits, phone click (tel:), email click.
Email : transactional via SendGrid / Mailgun (templates). Double opt-in pour newsletter.
Security : HTTPS obligatoire, CSP, rate limiting on forms (Captcha reCAPTCHA v3 or hCaptcha optional).
